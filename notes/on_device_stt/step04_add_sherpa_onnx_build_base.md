# Step 4: 文字化エンジン `sherpa_onnx` を入れて、アプリが壊れず起動することを確認する

## ひとことで言うと

音声を文字にするエンジン（`sherpa_onnx`）をアプリに追加し、
**「入れてもアプリがちゃんとビルドできて起動するか」「エンジンが読み込めるか」を確認する**ステップ。

このエンジンを実際に動かす（文字にする）にはモデルという大きなファイルが必要だが、その用意は後のステップ。
ここでは**入れても問題が起きないこと**だけを確かめる。

---

## このステップの目的

全体の流れの中で、このステップは「文字化エンジンを土台として入れる」部分にあたる。

```
マイク → 録音 → 区切り → 文字化 → メモに保存
                          ↑
                  ここで使うエンジンを今回入れる
```

ただしエンジンは**追加した瞬間にビルドが壊れやすい**性質があるため（理由は下の補足）、
実際に文字化する処理に進む前に、**「入れてもアプリが正常に動く」状態をまず作っておく**のがこのステップ。

### 補足: `sherpa_onnx` とは / なぜ壊れやすいのか

- **`sherpa_onnx`**: 音声から文字を起こす処理を担う部品（パッケージ）。中で AI を動かすための実行エンジン（onnxruntime）を使っている。
- 壊れやすいポイントは2つある。

**(A) Android: ネイティブ部品の重複**

- onnxruntime は端末ごとのネイティブな部品（`.so` ファイル等）を含むため、他の部品と**重複したりバージョンがぶつかったりするとビルドが失敗する**ことがある。
- 今回は重複の原因になる部品（`vad` パッケージ）をあえて入れていないので、特別な対策なしで通るはず。それを確認するのがこのステップの目的の1つ。

**(B) iOS: 埋め込み framework の「署名」（Step 3 で実際に遭遇）**

- sherpa_onnx は iOS 向けに、出来合いの部品（`sherpa_onnx.framework` / `objective_c.framework`）をアプリに**埋め込む**形で同梱する。
- 新しい iOS（この端末は iOS 26）では、埋め込む部品すべてに**コード署名**が必要で、署名がないと**インストール時に弾かれる**。
- 実際、Step 3 の iOS ビルドで `sherpa_onnx.framework : No code signature found`（署名が見つからない）というエラーに遭遇している。
  - そのときは古いビルドの残骸が原因だったが、**今回 sherpa_onnx を正式に追加すると、同じ署名エラーが本物として出る可能性がある**。
- 通常は CocoaPods が部品を取り込むと、ビルド時に開発用の証明書で自動的に署名される。ただし新しい Xcode/iOS の組み合わせでは、ここでつまずく事例が知られている。
- **このステップでの一番の注意点はここ**。Android の重複より、iOS の署名の方が今回は要注意。

---

## やること

### 1. `sherpa_onnx` を追加する

- `pubspec.yaml` に `sherpa_onnx: ^1.13.2` を1行追加する（Step 3 で入れた `record` の下）。
- 追加したら `flutter pub get` を実行する。

### 2. アプリがビルドできて起動することを確認する

- **Android**: 追加の対策（重複回避の設定など）を入れなくてもビルドが通り、起動することを確認する。
- **iOS**: ここが要注意。署名の問題は**ビルド時ではなくインストール時に出る**ため、「ビルドできた」で終わらせず、**実機にインストールして起動するところまで**確認する。
  - もし `No code signature found` などの署名エラーでインストールが弾かれたら、補足(B)の問題が現実化したということ。その場合は埋め込み framework が正しく署名されるよう設定（Embed & Sign など）を見直す。

### 3. 確認用の画面を作る（dev catalog にセクションを追加）

- 新規ファイル: `lib/pages/dev/catalog/sections/packages/packages_sherpa_onnx_section.dart`
- Step 3 で作った「Packages」セクションに、`sherpa_onnx` の確認アイテムとして追加する。

この画面で見せるもの:

- **エンジンの読み込みが成功するか**を表示する（`initBindings()` という読み込み処理を呼び、成功 / 失敗を画面に出す）。

> モデルがまだないので、ここで確認できるのは「エンジンが読み込めた」まで. 実際の文字化は後のステップ。

### 4. 設定ファイルの扱い

- Android のビルド設定 … 重複対策などの追加は不要（触らない）。
- iOS の対応バージョン（13.0）… 上げる必要はない（触らない）。
- iOS の **framework の署名設定** … 基本は触らずに済むはずだが、補足(B)の署名エラーが出た場合に限り、ここだけ見直す可能性がある。
